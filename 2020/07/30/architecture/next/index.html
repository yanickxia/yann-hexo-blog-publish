云原生的下一步

发表于 2020-07-30 更新于 2021-10-21 分类于 architecture
Powered by Time.Graphics

从时间线看，我们大致上可以从 2014 开始作为 微服务 元年，作为首先践行 微服务 的 SpringCloud 体系，从 2015 年，那时候还没有 CNCF 的体系诞生，因此形成了如下的两条路线：

  1. Spring Cloud + Docker: 应用侧自行搞定微服务的基础设施
  2. CNCF: 大包大揽的搞定基础设施

Spring Cloud 体系

Spring Cloud 体系优势是一体化程度搞，从Java开发体系中选择一些人出来就可以成为 Spring Cloud 的开发者，但是问题也很多。

  • 异构语言的接入
  • 研发要求的水涨船高

Cloud Native 体系

Cloud Native 体系立意就会比单纯的 SpringCloud 体系高一些，力争为所有的软件都提供统一的 微服务基础设施。在 2020 年的如今，我们的分布式系统基础设施已经完善了，Cloud OS 已经初具雏形，未来就是如何将这个 New OS 更好的服务于应用是我们值得思考的。
